# 화면 설계 (1) — 전체 화면 흐름

> 버전 v0.1 · 작성일 2026-06-14 · 용도: 화면 간 이동 구조(IA) + 진입/전환 정의
> 선행 문서: 01 설계문서 9장(화면 목록), 4장(기능 명세)

---

## 0. 읽는 법

- 본 문서는 **화면 간 이동 구조**만 정의한다. 개별 화면의 내부 레이아웃·컴포넌트는 다음 문서(05 화면설계_상세)에서 다룬다.
- 화면 코드: `S0`(메인) ~ `S5`(데이터 관리). 이후 문서·프로토타입에서 동일 코드 사용.

---

## 1. 화면 목록 (01 설계서 9장과 매핑)

| 코드 | 화면 | 역할 | 연결 기능 |
|---|---|---|---|
| S0 | 메인/대시보드 | 진입점, 기능 선택, 최근 분석 요약 | - |
| S1 | 단일 국가 진단 | 국가 입력 → 유사도 결과 + 판정 + 비용 | 기능1 |
| S2 | 권역 순위 | 권역 선택 → 순위표(카테고리 분해) | 기능2 |
| S3 | 설정 | 가중치/임계값/게이트/킬스위치/화이트리스트 | 서브(B) |
| S4 | 보고서 뷰어 | 요약/AI인사이트/상세/출처/조사일자/가중치 | 서브(A) |
| S5 | 데이터 관리 | 진출국 실적 입력, 조사 데이터 갱신·이력 | 데이터 자산 |

## 3. 진입·전환 정의

### 3.1 S0 메인 → 각 화면

| 트리거 | 이동 | 비고 |
|---|---|---|
| "국가 진단" 선택 | S0 → S1 | 기능1 진입 |
| "권역 순위" 선택 | S0 → S2 | 기능2 진입 |
| "설정" 선택 | S0 → S3 | |
| "데이터 관리" 선택 | S0 → S5 | |
| 최근 보고서 카드 클릭 | S0 → S4 | 저장된 보고서 바로 열람 |

### 3.2 S1 단일 국가 진단

| 트리거 | 이동 | 처리 |
|---|---|---|
| 국가 입력 + 분석 실행 | S1 내 결과 표시 | 기능1 STEP 1~4 수행 |
| 판정=이식가능/심층조사 | S1 결과 → "보고서 보기" | |
| "보고서 보기" | S1 → S4 | 결과 기반 보고서 생성·열람 |
| "가중치 바꿔 재실행" | S1 → S3(설정) → 복귀 | 또는 S1 내 간이 가중치 패널 |
| 데이터 부족 경고 | S1 → S5 | 조사 데이터 보완 유도 |

> **분석 실행 전 검증**: 대상국 조사 데이터(RESEARCH_SNAPSHOT)가 없으면 S5로 유도. 같은 권역 진출국이 없으면 "비교 대상 없음" 안내.

### 3.3 S2 권역 순위

| 트리거 | 이동 | 처리 |
|---|---|---|
| 권역 선택 + 실행 | S2 내 순위표 | 권역 내 전 국가에 기능1 루프 |
| 순위표 행 클릭 | S2 → S1(해당국 상세) 또는 S4 | 개별 국가 드릴다운 |
| "권역 보고서" | S2 → S4 | 권역 종합 보고서 |

### 3.4 S3 설정 → 결과 재계산

| 트리거 | 이동 | 처리 |
|---|---|---|
| 가중치/임계값 저장 | S3 내 저장 | **새 ruleset 버전 생성**(기존 수정 금지) |
| "저장 후 재분석" | S3 → S1 또는 S2 | 직전 분석을 새 룰셋으로 재실행 |

> **재현성 주의**: 설정 변경은 새 ruleset을 만든다. 과거 보고서(S4)는 그때의 weight_snapshot으로 그대로 재현된다(설계서 8장 리스크7).

### 3.5 S4 보고서 뷰어

| 트리거 | 이동 | 비고 |
|---|---|---|
| "원본 분석으로" | S4 → S1/S2 | 분석 화면 복귀 |
| "이 보고서 가중치로 재실행" | S4 → S3 | 보고서의 weight_snapshot을 설정에 로드 |
| 출처 항목 클릭 | S4 내 출처 패널 | 공식/Tier 표시 확인 |

### 3.6 S5 데이터 관리

| 트리거 | 이동 | 처리 |
|---|---|---|
| 진출국 실적 입력·수정 | S5 내 | ENTRY_RECORD 갱신 |
| 조사 데이터 갱신 | S5 내 | 새 RESEARCH_SNAPSHOT 생성(조사일자) |
| "이 국가 진단하기" | S5 → S1 | 조사 직후 바로 분석 |

## 4. 핵심 사용자 여정 (User Journey)

### 여정 A — "이 나라 진출하면 얼마 들까?" (기능1 중심)
```
S0 → S1(체코 입력) → [분석] → S1 결과(이식가능 75점, 비용 55%)
   → S4(근거 보고서 확인) → [가중치 의심] → S3(시스템 비중↑)
   → S1 재실행 → S4(재보고서)
```

### 여정 B — "어느 나라부터 갈까?" (기능2 중심)
```
S0 → S2(유럽 선택) → [분석] → S2 순위표(1.체코 2.헝가리 ...)
   → 1위 행 클릭 → S1(체코 상세) → S4(보고서)
```

### 여정 C — "새 나라 조사부터" (데이터 우선)
```
S0 → S5(폴란드 조사 데이터 입력) → [조사완료]
   → "이 국가 진단하기" → S1(폴란드) → S4
```

## 5. 화면 간 공통 규칙

1. **조사일자 표시**: S1·S2·S4는 사용된 조사 데이터의 조사일자를 항상 노출(진출국 스냅샷 vs 대상국 현행 시점 격차).
2. **가중치 룰셋 표시**: 분석 결과 화면(S1·S2·S4)은 현재 적용 중인 ruleset 이름·버전을 항상 표시.
3. **킬스위치 우선 경고**: 판정이 BLOCKED(킬스위치 작동)면 점수와 무관하게 최상단 경고로 표시.
4. **공식 출처 갭 표시**: Tier2·3로 채운 항목은 결과·보고서에서 "공식 출처 미확보" 뱃지 노출.
5. **뒤로가기 일관성**: 모든 하위 화면은 S0로 복귀 가능. 분석 결과는 휘발되지 않고 저장(SIMILARITY_RESULT) 후 재진입 가능.

# 화면 설계 (1) — 전체 화면 흐름

> 버전 v0.2 · 작성일 2026-06-18 · 용도: 화면 간 이동 구조(IA) + 진입/전환 정의
> 선행 문서: `ui_design_guide.md`(화면별 디자인·기능 정의)
> 산출물: `design/html/<화면ID>/` 아래 `code.html`(목업) + `screen.png`(렌더 이미지)

---

## 0. 읽는 법

- 본 문서는 **화면 간 이동 구조(IA)**만 정의한다. 개별 화면의 내부 레이아웃·컴포넌트·디자인 토큰은 `ui_design_guide.md`와 `design/html/guide_md/DESIGN.md`에서 다룬다.
- 화면 코드 체계:
  - **M1~M7** — 풀사이즈 화면(Main/Screen)
  - **P1~P4** — 지도/화면 위에 뜨는 팝업·모달(Popup)
- 각 화면의 목업은 `design/html/<소문자ID>/code.html`(예: `design/html/m1/code.html`)에 있으며, 동일 폴더의 `screen.png`로 렌더 결과를 확인할 수 있다.
- HTML 목업은 각각 독립 파일이라 실제 하이퍼링크로 연결되어 있지 않다. 아래 전환 관계는 `ui_design_guide.md`의 "기능" 정의를 구현 기준으로 본 것이다.

## 2. 상단 네비게이션 구조 (IA)

전 화면 공통으로 **상단바만** 존재한다(좌·우 사이드바 없음). M1 기준 메뉴 트리:

```
상단 메뉴바
├─ 메인 .................................. M1
├─ 상세 (드롭다운)
│   ├─ 권역 상세 ......................... M2
│   ├─ 국가 상세 ......................... M3
│   └─ 국가 전체 보기 ................... M7
├─ 보고서 (드롭다운)
│   ├─ 권역 보고서 ....................... M4
│   └─ 국가 보고서 ....................... M5
├─ 설정 (드롭다운)
│   └─ 가중치 설정 ....................... M6
└─ 언어 전환 (최상단 우측)
    ├─ 태극기 → 한국어
    └─ 영국기 → 영어 (전체 한글↔영문 변환)
```

## 3. 전체 흐름도

```
                               ┌──────────────────────┐
                               │   M1 메인 / 세계지도   │  ◀── 진입점
                               └───────────┬──────────┘
        ┌──────────────┬──────────────┬────┴─────┬──────────────┬─────────────┐
        ▼              ▼              ▼          ▼              ▼             ▼
   권역 마크 클릭   국가 마크 클릭  진단패널      메뉴: 상세      메뉴: 설정    메뉴: 상세
        │              │          [상세보기]      │              │         [국가 전체]
        ▼              ▼              ▼          │              ▼             ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐     │         ┌─────────┐  ┌─────────┐
   │ P1 권역 │    │ P2 국가 │    │ P3 진행 │     │         │ M6 설정 │  │ M7 국가 │
   │ 요약팝업│    │ 요약팝업│    │  팝업   │     │         │ (가중치)│  │ 리스트  │
   └────┬────┘    └────┬────┘    └────┬────┘     │         └─────────┘  └────┬────┘
        │ 상세보기      │ 상세보기      │ 보고서보기 │                          │ 행 클릭
        ▼              ▼          (권역/국가 분기)│                          ▼
   ┌─────────┐    ┌─────────┐         │          │                     ┌─────────┐
   │ M2 권역 │    │ M3 국가 │         │          │                     │ M3 국가 │
   │ 상세정보│    │ 상세정보│         │          │                     │ 상세정보│
   └──┬───┬──┘    └──┬───┬──┘         │          │                     └─────────┘
      │   │          │   │            │          │
[시뮬]│   │[보고서] [시뮬]│   │[보고서] │          │
      ▼   │          ▼   │            │          │
  ┌─────────┐ │      ┌─────────┐ │    │          │
  │ P3 진행 │ │      │ P3 진행 │ │    │          │
  │  팝업   │ │      │  팝업   │ │    │          │
  └────┬────┘ │      └────┬────┘ │    │          │
       │      ▼           │      ▼    ▼          │
       │  ┌─────────┐     │  ┌─────────────┐     │
       └─▶│ M4 권역 │     └─▶│ M5 국가      │◀────┘ (P3 국가 분기 / M3 보고서)
          │ 보고서  │        │ 보고서       │
          └─────────┘        └─────────────┘
   (P3 권역 분기 / M2 보고서)

      ┌─────────┐  [실행]   ┌─────────┐
      │ P4 룰셋 │ ───────▶ │ P3 진행 │   (가중치 룰셋 선택 → 진단 실행)
      │  팝업   │           │  팝업   │
      └─────────┘           └─────────┘
```

핵심 규칙:
- **P3(프로그레스)는 권역/국가 구분 파라미터로 분기** — 권역이면 완료 후 M4, 국가면 M5로 이동.
- **M2/M3의 [보고서] 버튼은 P3를 거치지 않고** 각각 M4/M5로 직행(이미 산출된 보고서 열람).

## 4. 진입·전환 정의

### 4.1 M1 메인 / 세계지도

| 트리거 | 이동 | 전달 파라미터 / 처리 |
|---|---|---|
| 지도 내 **권역** 마크/영역 선택 | M1 → P1 | 권역 확대 애니메이션 + 권역 요약 카드 팝업 |
| 지도 내 **국가** 마크 선택 | M1 → P2 | 국가 요약 카드 팝업 |
| 진단 프로그레스 패널 [상세보기] | M1 → P3 | 진행 중인 국가/권역 |
| 진단 프로그레스 패널 [보고서] | M1 → M4 또는 M5 | 진행률 100%일 때만 활성. 권역=M4 / 국가=M5 |
| 메뉴: 상세 → 권역상세 / 국가상세 / 국가전체 | M1 → M2 / M3 / M7 | |
| 메뉴: 보고서 → 권역 / 국가 | M1 → M4 / M5 | |
| 메뉴: 설정 → 가중치 설정 | M1 → M6 | |
| 언어 전환 (태극기 / 영국기) | M1 내 | 전체 한글 ↔ 영문 변환 |

### 4.10 M6 설정 (가중치)

| 트리거 | 처리 |
|---|---|

|---|---|---|

| 테이블 행 선택 | M7 → M3 | 해당 국가 상세 |

## 5. 핵심 사용자 여정 (User Journey)

### 여정 A — "어느 권역부터 갈까?" (권역 quick-win 탐색)
```
M1(지도) → 권역 마크 클릭 → P1(권역 요약) → [상세보기] → M2(권역 상세)
   → [시뮬레이션] → P3(진행) → [보고서보기] → M4(권역 보고서)
```

### 여정 B — "이 나라 진출하면 어떨까?" (국가 진단)
```
M1(지도) → 국가 마크 클릭 → P2(국가 요약) → [상세보기] → M3(국가 상세)
   → [시뮬레이션] → P3(진행) → [보고서보기] → M5(국가 보고서)
```

### 여정 C — "가중치를 바꿔 다시 본다" (룰셋 조정 후 재실행)
```
M6(설정) 또는 P4(룰셋 팝업) → 가중치/룰셋 조정 → [실행]
   → P3(진행) → [보고서보기] → M5(또는 M4)
```

## 6. 화면 간 공통 규칙

1. **상단바 전용 레이아웃**: 모든 화면은 상단 메뉴바만 가지며 좌·우 사이드바가 없다.
2. **한/영 전환**: 최상단 우측 국기 버튼으로 전체 한글 ↔ 영문 변환.
3. **프로그레스 패널**: M1 최상단 우측에 카드형 진단 프로그레스 패널을 노출/비노출 토글. 국가·권역명, 진행률, [상세보기]·[보고서] 버튼 포함.
4. **보고서 PDF**: M4·M5는 보고서 ID·생성일시 표시와 PDF 다운로드 버튼을 항상 제공.
5. **룰셋·조사일자 표시**: 분석 결과/보고서 화면은 적용 룰셋과 조사 기준 시점을 노출(M5는 비교국 스냅샷 일자 포함).
6. **구분 파라미터 일관성**: 시뮬레이션·프로그레스(P3)는 항상 `구분(권역/국가)`과 선택값을 파라미터로 전달하여 결과 보고서(M4/M5) 분기를 결정.

## 0. 한 줄 요약

진출국(base)을 기준으로 예상 진출국(target)의 항목별 정규화 값을 비교해, **항목 → 카테고리 → 종합** 순으로 유사도(0~100)를 내고, 킬스위치·게이트·임계값으로 진출 가능 여부를 판정한다. v2는 데이터 타입별 정규화를 정교화하고 신뢰도(conf)를 항목 단위로 차등 적용한다.

## 2. 항목 유형별 유사도 계산 (v2 정교화)

### 2.1 연속형 — 두 갈래로 분리 (v2 핵심 변경)

문서 Part 3.1의 "스케일 큰 값 vs 비율" 구분을 채택했다.

**(a) 스케일 큰 값** (판매대수, 시장규모) → log10 후 min-max 정규화
```
norm(v) = (log10(v) − log10(min)) / (log10(max) − log10(min))
유사도 = 100 × (1 − |norm(A) − norm(B)|)
```
- 적용: MKT_CAR_SALES(유니버스 10만~400만), MKT_AF_SIZE(10억~1000억 USD)
- 이유: 영국 202만 vs 폴란드 60만처럼 절대값 차가 큰 항목을 직접 빼면 왜곡. 로그 공간에서 비교.

**(b) 비율(%)** → 직접 거리
```
유사도 = max(0, 100 × (1 − |A−B| / scale))
```
- scale = "완전히 다르다"고 볼 차이. 항목별: 세율 30%p, 침투율 100%, 신차/중고배수 6
- 예: 법인세 25% vs 19% → |6|/30 → 80점

**(c) 일수/개월** → 전용 scale (연체분류 180일, 인허가 365일, 대출기간 60개월)

**(d) 객체형** (여러 지표, 예: 평균금리 {new, prime, subprime}) → 공통 키별 계산 후 평균

### 2.2 순서형 (Ordinal) — v2 신규

등급형 코드값을 레벨 거리로 계산한다.
```
유사도 = 100 × (1 − |level(A) − level(B)| / maxLevel)
```
- 적용: 디지털 성숙도(LOW0/MEDIUM1/MEDIUM_HIGH2/HIGH3), 벤더 락인(LOW0~HIGH2)
- 예: HIGH(3) vs MEDIUM_HIGH(2) → |1|/3 → 67점 (단순 불일치 0점보다 정확)

### 2.3 범주 (Categorical)
- **단일 코드값**: 일치 100, 불일치 시 토큰 부분일치 (HIGH vs MEDIUM_HIGH → 공통 토큰)
- **배열** (Top리스트·벤더): 자카드 = 교집합/합집합 × 100
  - 예: 브랜드 [VW,BMW,FORD,KIA,AUDI] vs [TOYOTA,SKODA,VW,KIA,HYUNDAI] → 2/8 → 25점

### 2.4 이진 (Binary)
규제 충족 여부. 같으면 100, 다르면 0. 실제 진출 차단은 킬스위치 선검증(3장)에서 별도 처리.

### 2.5 참고 (Reference)
유사도 계산 제외. 점수 영향 없음(경쟁사 금리 Range, 시장 성장률 등). 보고서에만 표시.

## 3. 신뢰도(conf) — v2 정교화

문서 Part 3.2의 항목별 conf를 채택. **두 단계로 곱한다.**

```
항목 conf = 항목별 기본 conf(itemConf 테이블) × 데이터 신뢰도계수(낮은쪽)
```

- **항목별 기본 conf**: 데이터 특성상 본질적 정확도. 라이선스 0.95, 침투율 0.7, 시스템(T5) 0.3~0.4
- **데이터 신뢰도계수**: 실제 채운 출처의 Tier (두 나라 중 낮은 쪽 채택, 보수적)
- 예: 시스템 벤더 항목 = 기본 0.4 × (양국 Tier2 = 0.8) = 0.32 → 점수 기여 대폭 축소

**효과**: 출처가 부실하거나(낮은 Tier) 데이터 특성상 부정확한(시스템 등) 항목은 자동으로 영향력이 줄어든다.

## 4. 판정 파이프라인 (순서 중요)

```
STEP 1. 킬스위치 선검증
   target의 is_killswitch 항목 중 value_normalized === false 있으면
   → verdict = BLOCKED (점수와 무관)

STEP 2. 카테고리 점수 (conf 가중 평균)
   카테고리 점수 = Σ(항목유사도 × conf) / Σ(conf)
   ※ 공백(null) 항목 제외

STEP 3. 시스템 게이트
   SYSTEM 점수 < 50 → verdict = DEEP_RESEARCH (종합 높아도 이식 불가)

STEP 4. 종합 점수 (카테고리 가중)
   종합 = Σ(카테고리점수 × 카테고리가중치) / Σ(사용된 가중치)
   가중치: MARKET 0.25 / REGULATORY 0.25 / FINANCIAL 0.20 / SYSTEM 0.30
   ※ 결측 카테고리는 분모에서 제외 보정

STEP 5. 유효신뢰도 (v2 신규, 문서 3.3)
   유효신뢰도 = Σ(카테고리가중치 × conf) / Σ(카테고리가중치)
   < 0.6 → "저신뢰" 플래그 → 단독 의사결정 금지 경고

STEP 6. 최종 판정
   BLOCKED          : 킬스위치 작동
   TRANSPLANTABLE   : 게이트 통과 AND 종합 ≥ 70
   DEEP_RESEARCH    : 그 외
```

---

## 5. 비용 추정 (구간표)

판정이 TRANSPLANTABLE일 때만. 종합 유사도 → 비교국 대비 비용.

| 종합 유사도 | 비교국 대비 비용 | 절감률 |
|---|---|---|
| 90 이상 | 35% | 65% |
| 80~90 | 48% | 52% |
| 70~80 | 62% | 38% |
| 70 미만 | 산출 안 함 | — |

## 6. 데이터 공백(GAP) 처리

`value_normalized = null` 항목은 계산에서 제외. 점수는 "채워진 항목만의 잠정치".
- 공백 많은 카테고리(특히 SYSTEM·경쟁사)는 신뢰도 낮음 → 보고서에 공백 개수 표시
- 유효신뢰도 지표가 이 부실을 점수화 → 0.6 미만이면 경고
- 공백 = 현지 1차 조사 발주 리스트

## 7. 적용 예시 (영국 기준, 폴란드 대상 — v2 결과)

> base = United Kingdom (진출국 가정), target = Poland

| 카테고리 | 유사도 | 가중치 | 비고 |
|---|---|---|---|
| MARKET | 47.7 | 25% | 영국 리테일 vs 폴란드 법인리스 |
| REGULATORY | 49.8 | 25% | EU 공통 일부 일치, 세제·라이선스 상이 |
| FINANCIAL | 37.3 | 20% | PCP vs 리스 구조 차이 |
| SYSTEM | 34.2 | 30% | **게이트 50 미달 → 이식 불가** |

- 킬스위치: 전부 통과
- 유효신뢰도: 0.67 (임계 0.6 초과 → 정상)
- 종합: **42.1** → 판정 **DEEP_RESEARCH**
- 데이터 공백 26개

**v1 대비 변화**: 종합 38.6 → 42.1. log 변환·순서형·conf 차등으로 시스템 점수가 25 → 34로 상승. 단 게이트(50) 여전히 미달.

**해석**: 영국·폴란드는 시장·시스템 구조가 달라 시스템 이식 부적합. 공백이 절반에 가까워 시스템 항목 보강 후 재검증 필요.

## 8. 한계 및 주의

1. **공백 비중이 점수 좌우**: 양국 모두 공백 26개로 점수는 잠정치.
2. **scale·유니버스 민감도**: 연속형 scale과 log 유니버스(min/max) 설정이 결과에 영향. 현재값은 초안, 항목별 튜닝 필요.
3. **방향성**: 유사도는 대칭이나, 의미 있는 건 "진출국(base) → 예상진출국(target)" 방향.
4. **conf 보수 채택**: 두 나라 중 낮은 conf를 쓰므로 한쪽 데이터가 약하면 신뢰도 낮게 반영.
5. **침투율 정의 미통일**: 문서 Part 3.4의 침투율 정의 통일(분모=신차판매, 분자=금융+리스 대수)은 아직 데이터에 미반영. 영국(FLA 직접) vs 폴란드(리스 가치환산)는 정의가 달라 현재 비교에 오차 존재 → 향후 정형화 과제.

## 9. 함수 구조 (similarity.js v2)

| 함수 | 역할 |
|---|---|
| `simPct(a,b,scale)` | 비율 직접 거리 |
| `simLog(a,b,uni)` | 스케일 큰 값 log10→min-max |
| `simOrdinal(a,b,map)` | 순서형 레벨 거리 |
| `simJaccard(a,b)` | 집합 자카드 |
| `simCategory(a,b)` | 범주 단일값 일치/부분일치 |
| `itemSimilarity(item,other)` | 항목 1개 유사도 (유형 분기) |
| `itemConf(item,other)` | 항목 conf = 기본 conf × 데이터 계수 |
| `compareCountries(target,base)` | 메인 파이프라인 |
| `costFromSimilarity(sim)` | 비용 구간표 |
| `printReport(result)` | 콘솔 리포트 |

브라우저: `window.SimilarityEngine.compareCountries(target, base)`

## 10. 설정 가능 항목 (RULESET)

대시보드 설정 화면에서 변경 가능:
- `categoryWeights`: 카테고리 가중치 (현재 시장25/규제25/금융20/시스템30)
- `thresholds`: entry 70 / systemGate 50 / lowConfidence 0.6
- `pctScale`, `logUniverse`, `dayScale`, `monthScale`: 정규화 스케일
- `itemConf`: 항목별 기본 신뢰도
- `ordinalMap`: 순서형 레벨 정의

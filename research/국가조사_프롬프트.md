한국 캐피탈사의 진출 검토용이며, 영국·폴란드와 동일 항목·동일 단위로 비교 가능하게 정리하는 것이 목적이다.

- 통화 금액은 {현지통화} 원값 + USD 환산 병기

## C. 조사 후 정규화 가이드 (코드값 표준)

조사 결과를 JSON(value_normalized)으로 적재할 때, 아래 표준 코드값을 사용한다.
(semantic_matrix.js에 정의된 값과 일치시켜야 유사도 계산 가능)

### 연속형 (숫자 그대로)
판매대수·시장규모·침투율·세율·금리·기간·LTV → 숫자 또는 {min,max,typical}. 통화는 USD 환산 병기.

### 범주형 (표준 코드값 사용)
| 항목 | 허용 코드값 |
|---|---|
| 라이선스 종류 | FCA_CONSUMER_CREDIT / KNF_LENDER_REG / EU_PASSPORT / (신규 시 추가) |
| 법인형태 | LTD_PLC / SPZOO_SA / GMBH_AG / (신규) |
| 개인정보보호 | UK_GDPR / EU_GDPR / PDPA / CCPA / (신규) |
| 국외이전 | EU_FREE / ADEQUACY_SCC / RESTRICTED / LOCALIZED |
| 충당금 | IFRS9 / LOCAL_GAAP / IFRS9_OR_LOCAL |
| 신용정보(CB) | PRIVATE_3BUREAU / BIK_CENTRAL / STATE_REGISTRY / NONE |
| 추심 규제 | FCA_CONC / CIVIL_COURT_BAILIFF / COURT_ONLY / SELF_HELP |
| 차량회수 | CCA_PROTECTED_GOODS / LEASE_OWNERSHIP / FREE_REPO / COURT_ORDER |
| 의무보험 | MANDATORY / MANDATORY_PLUS / OPTIONAL |
| 캡티브 구분 | CAPTIVE_DOMINANT_NEW / BANK_LEASING_DOMINANT / CAPTIVE_DOMINANT / INDEPENDENT |
| 솔루션 유형 | PACKAGE_SAAS_SI / PACKAGE_SI_CLOUD / SAAS_ONLY / FULL_INHOUSE |
| 디지털 성숙도 | LOW / MEDIUM / MEDIUM_HIGH / HIGH |

> 새 코드값이 필요하면 semantic_matrix.js에 해당 쌍의 유사도(0~1)를 추가해야 함.
> 추가 안 하면 토큰매칭 fallback으로 계산되어 부정확.

### 이진형 (킬스위치)
외국인지분·금리상한·외환송금·배당송금·데이터현지화 → true(진출가능/충족) / false(차단)

## D. 조사 → 적재 워크플로우

```
1. B의 프롬프트로 {국가명} 조사 → 69개 항목 값+출처+티어
2. country_{코드}.json 생성 (value_raw, source_detail, source_tier, research_detail)
3. 정규화: C 가이드로 value_normalized 채움 (코드값 표준 준수)
4. semantic_matrix.js에 신규 코드값 쌍 유사도 추가
5. node run_v3.js 로 베이스(영국 등) 대비 점수 계산
6. node gen_report.js 로 상세보고서 생성
```

## E. 베이스/타깃 국가 구성 (현재 계획)

- **베이스(진출국 가정)**: 영국, 미국, 호주
- **예상 진출국**: 폴란드, 스페인, 뉴질랜드, 인도
- 완료: 영국, 폴란드 / 미완: 미국, 호주, 스페인, 뉴질랜드, 인도
- 권역별 비교 시 같은 권역 베이스 사용 (유럽 타깃 → 영국 베이스)
